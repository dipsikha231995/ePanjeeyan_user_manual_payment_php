One of the goal of this android application is to make deed registration. Therefore, to make this experience as intuitive as possible, the whole deed registration process has been divided into four distinct, but related stages. The stages are as follows:

  1. Appointment Form
  2. Upload Documents
  3. Confirm Submit
  4. Make Payment

Stage 1 is for "Appointment Form". Here, a user has to provide his/her personal information and also information about the applicant type, appointment date, office for registration, deed category and sub deed category. Some of the data are needed to be filled while some others are in dropdowns for the user to choose from.

Only upon completing a particular stage, a user can move forward to a new stage. Likewise, to get to the final stage, all the previous stages must be completed first. All the stages are therefore needed to be filled with valid input in order to complete the registration process successfully.

Stage 2 is for "Upload Documents". Here the user needs to upload three types of documents, they are, Address proof, Age proof and Identity proof. The documents can be either pdf files or image files.

After Stage 2, comes "Confirm Submit", viz. Stage 3. This stage let users verify the data they have provided and make appropriate correction to the data if found incorrect. Now, at this point of time, the users should be able to move back and forth between stages. Once, everything is verified the user can hit "Confirm Submit" button to make an appointment. This will display the appointment number alloted to the user along with date and time and fee needed for the registration.

On completing stage 3, the user arrives at the final stage, "Make Payment". Here, the user has to fill in personal information like, department tax id, payer name, mobile number, pan number, addresses etc. As only a few fields have "*" mark on them so, apart form those "*"-marked mandatory fields, the remaining fields are optional here.

At e-GRAS Payment Gateway, the user needs to select the bank. A user can make online payments only at those banks listed in e-GRAS' website. If everything is alright then a GRN No. for that particular transaction will be generated. This GRN No. must be noted down as this number will be required to search for and identify the challan of a successufull transaction. After that, the user should be able to make the payment at the bank's respective payment portal.

Once, the payment is made, the user should be able to see a summary of that particular transaction. The summary includes information like, transaction's success status, date and time, GRN no, payer's name, office etc.
