One of the two goals of this android application is to make marriage registration. Therefore, to make this experience as intuitive as possible, the whole marriage registration process has been divided into five distinct, but related stages. The stages are as follows:

  1. Marriage Details
  2. Bride Details
  3. Groom Details
  4. Confirm Submit
  5. Make Payment

Stage 1 is for "Marriage Details". Here, an applicant has to provide his/her name, email (optional), mobile no. and also information about the appointment date, marriage type and office for registration. Some of the data are needed to be filled while some others are in dropdowns for the user to choose from.

Stage 2 is for "Bride Details". Here, an applicant has to provide bride's personal information like name, age, occupation, father's name, marital status, present and permanent address details.

Only upon completing a particular stage, a applicant can move forward to a new stage. Likewise, to get to the final stage, all the previous stages must be completed first. All the stages are therefore needed to be filled with valid input in order to complete the registration process successfully.

In Stage 3 i.e., for "Groom Details", an applicant has to provide groom's personal information like name, age, occupation, father's name, marital status, present and permanent address details.

After Stage 3, comes "Confirm Submit", viz. Stage 4. This stage let the applicants verify the data they have provided and make appropriate correction to the data if found incorrect. Now, at this point of time, the applicants should be able to move back and forth between stages. Once, everything is verified the applicants can hit "Confirm Submit" button to make an appointment. This will display the appointment number alloted to the applicants along with date and time and fee needed for the registration.

On completing stage 4, the applicant arrives at the final stage, "Make Payment". Here, the applicant has to fill in personal information like, department tax id, payer name, mobile number, pan number, addresses etc. As only a few fields have "*" mark on them so, apart form those "*"-marked mandatory fields, the remaining fields are optional here.

At e-GRAS Payment Gateway, the user needs to select the bank. A user can make online payments only at those banks listed in e-GRAS' website. If everything is alright then a GRN No. for that particular transaction will be generated. This GRN No. must be noted down as this number will be required to search for and identify the challan of a successufull transaction. After that, the user should be able to make the payment at the bank's respective payment portal.

Once, the payment is made, the user should be able to see a summary of that particular transaction. The summary includes information like, transaction's success status, date and time, GRN no, payer's name, office etc.
