The goal of this android application is to make payments and generate challans online. Therefore, to make this experience as intuitive as possible, the whole process has been divided into four distinct, but related stages. The stages are as follows:

  1. Department Details
  2. Scheme Details
  3. Payer Details
  4. Data Verification

Stage 1 is for "Department Details". Here, a user has to provide information about the department, payment type, district, office, and the year and period the payment is going to be made for. Each input data is presented in the form of dropdowns for the user to choose form.

Only upon completing a particular stage, a user can move forward to a new stage. Likewise, to get to the final stage, all the previous stages must be completed first.

Stage 2 is for "Scheme Details". For every department and office, there are different schemes for which the offices have to pay tax to the Government. On the basis of what the user has chosen for department and office in stage 1, the schemes will be listed here. The user has to fill in the amount for the scheme(s) he/she is paying for.

After Stage 2, comes "Payer Details", viz. Stage 3. Here, the user has to fill in personal information like, department tax id, payer name, mobile number, pan number, addresses etc. As only a few fields have "*" make on them so, apart form those "*"-marked mandatory fields, the remaining fileds are optional here.

On completing stage 3, the user arrives at the final stage, "Data Verification". This stage let users verify the data they have provided and make appropriate correcetion to the data if found incorrect. Now, at this point of time, the users should be able to move back and forth between stages. Once, everything is verified the user can hit "Submit" button to make the payment. This will take the user to e-GRAS Payment Gateway.

At e-GRAS Payment Gateway, the user needs to select the bank. A user can make online payments only at those banks listed in e-GRAS' website. If everything is alright then a GRN No. for that particular transaction will be generated. This GRN No. must be noted down as this number will be required to search for and identify the challan of a successufull transaction. After that, the user should be able to make the payment at the bank's respective payment portal.

Once, the payment is made, the user should be able to see a summary of that particular transaction. The summary includes information like, transaction's status(Success, Pending or Fail), date and time, GRN no, payer's name, office etc. In case of a successfull transaction, the user should be able the download the challan.
